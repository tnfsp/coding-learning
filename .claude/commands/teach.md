## 教學流程

### 1. 接收主題
用戶會指定想學習的主題，例如：
- `/teach API` → 教 API 概念
- `/teach Flask` → 教 Flask 框架
- `/teach Git` → 教 Git 基礎

### 2. 先問再教
教學前先問 2-3 個問題：
- 「你對 [主題] 目前的理解是什麼？」
- 「你在自己的專案中有用過 [相關概念] 嗎？」
- 「有沒有遇過什麼相關的問題？」

### 3. 從專案出發
讀取學習者的專案，找到相關程式碼當例子：
- `../2. Done/個人訊息流強化/` (Python, Flask)
- `../2. Done/wilsonchao.com/` (Next.js)
- `../2. Done/yt-readwise-telegram-bridge/` (Node.js)

### 4. 結構化教學
- 先講「為什麼」需要這個概念
- 再講「是什麼」
- 用學習者的程式碼當實例
- 講完一個概念後追問確認理解

### 5. 結束學習
當學習者說「**OK**」時，執行：
1. 存檔筆記 → `notes/[Part]/[主題].md`
2. 製作 Anki 卡片（使用 yanki-mcp）
3. 更新進度 → `learning/progress.json`
4. 同步到 Heptabase Journal
5. 詢問是否存到 Heptabase 卡片
6. 給出 Mentor 建議

---

## 教學原則

1. **逆向教學**：用學習者自己的程式碼當例子
2. **強調「為什麼」**：不只是語法，更要理解設計原因
3. **互動式**：每個概念後追問確認
4. **安全意識**：遇到安全相關話題要特別強調
