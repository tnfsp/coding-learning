# AI 時代 Coding 學習教師 - 產品需求文件 (PRD)

> 由 `/concept` 維護

---

## 專案概述

### 專案目標

協助一位已有 Vibe Coding 經驗的學習者，系統性地**填補知識空白**，從「能跑就好」進階到「知道為什麼」。

核心問題：**在 AI 輔助編程時代，人類需要具備的最低限度 Coding 技能是什麼？**

### 核心價值

1. **逆向學習法**：不是從零學程式，而是從已完成的專案逆向理解原理
2. **AI 協作導向**：學習重點是「如何更好地與 AI 協作」，而非「取代 AI 自己寫」
3. **實用主義**：只學「心臟外科醫師」等級的知識——精準、關鍵、能做決策

---

## 目標用戶

### 用戶畫像

- **身份**：心臟外科醫師，非資訊背景
- **Vibe Coding 經驗**：
  - 已完成 Next.js 個人網站
  - 已完成 Python Flask + Telegram Bot 自動化
  - 已部署到 Zeabur 雲端
  - 使用 Git 版本控制
- **現況**：能用 AI 完成專案，但不太理解程式碼運作原理
- **目標**：成為能與 AI 有效協作的「心臟外科醫師型」使用者

### 使用場景

1. **學習時間**：零碎時間、工作空檔
2. **學習方式**：互動式教學，邊問邊學
3. **輸出需求**：Anki 卡片（記憶）、筆記（參考）、Heptabase（知識圖譜）

## 非功能需求

### 教學原則

1. **先問再教**：了解學習者已知什麼，避免重複
2. **連結實務**：用學習者自己的專案當例子
3. **強調「為什麼」**：不只是語法，更要理解設計原因
4. **安全意識**：特別強調安全相關知識

### 互動風格

- 對話式，非講課式
- 每個概念教完後追問確認理解
- 適時給予鼓勵和進度回饋

## 課綱大綱

### Part 1：程式世界觀（心智模型）
- 電腦如何執行程式
- 前端 vs 後端 vs 資料庫
- API 是什麼

### Part 2：看懂你的專案（逆向理解）
- Python 基礎：import、函數、類別
- JavaScript 基礎：變數、函數、物件
- 框架概念：Flask、Next.js
- 環境變數與 secrets

### Part 3：AI 協作技能
- 如何描述需求
- 如何讀懂 AI 輸出
- 常見錯誤類型與除錯

### Part 4：安全與風險意識
- API key 管理
- 常見安全漏洞
- 權限與存取控制

### Part 5：工具鏈素養
- Git 基本概念
- 終端機基礎
- 套件管理（npm/pip）
